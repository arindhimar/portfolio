Understanding Bubble Sort and Its Optimizations in C++

Basic Bubble Sort Algorithm

void bubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
            }
        }
    }
}

Explanation

The basic Bubble Sort algorithm sorts an array by repeatedly comparing adjacent elements and swapping them if they are in the wrong order. This process continues until the entire array is sorted. The algorithm performs nâˆ’1 passes, where n is the number of elements in the array. The time complexity of this algorithm is O(n2), which makes it inefficient for large datasets.

Dry Run

Let's take an example: arr = {64, 34, 25, 12, 22, 11, 90}.

  • Initial Array: {64, 34, 25, 12, 22, 11, 90}
  • First Pass:
    • Compare 64 and 34 -> swap -> {34, 64, 25, 12, 22, 11, 90}
    • Compare 64 and 25 -> swap -> {34, 25, 64, 12, 22, 11, 90}
    • Compare 64 and 12 -> swap -> {34, 25, 12, 64, 22, 11, 90}
    • Compare 64 and 22 -> swap -> {34, 25, 12, 22, 64, 11, 90}
    • Compare 64 and 11 -> swap -> {34, 25, 12, 22, 11, 64, 90}
    • Compare 64 and 90 -> no swap -> {34, 25, 12, 22, 11, 64, 90}
  • Second Pass:
    • Compare 34 and 25 -> swap -> {25, 34, 12, 22, 11, 64, 90}
    • Compare 34 and 12 -> swap -> {25, 12, 34, 22, 11, 64, 90}
    • Compare 34 and 22 -> swap -> {25, 12, 22, 34, 11, 64, 90}
    • Compare 34 and 11 -> swap -> {25, 12, 22, 11, 34, 64, 90}
    • Compare 34 and 64 -> no swap -> {25, 12, 22, 11, 34, 64, 90}
  • Third Pass:
    • Compare 25 and 12 -> swap -> {12, 25, 22, 11, 34, 64, 90}
    • Compare 25 and 22 -> swap -> {12, 22, 25, 11, 34, 64, 90}
    • Compare 25 and 11 -> swap -> {12, 22, 11, 25, 34, 64, 90}
    • Compare 25 and 34 -> no swap -> {12, 22, 11, 25, 34, 64, 90}
  • Fourth Pass:
    • Compare 12 and 22 -> no swap -> {12, 22, 11, 25, 34, 64, 90}
    • Compare 22 and 11 -> swap -> {12, 11, 22, 25, 34, 64, 90}
    • Compare 22 and 25 -> no swap -> {12, 11, 22, 25, 34, 64, 90}
  • Fifth Pass:
    • Compare 12 and 11 -> swap -> {11, 12, 22, 25, 34, 64, 90}

The array is now sorted, and Bubble Sort completes its execution.

Pros and Cons

Pros:

  • Simple to implement and understand.

Cons:

  • Inefficient for large datasets due to its O(n2) time complexity.

Optimized Bubble Sort by Early Termination

void optimizedBubbleSort(int arr[], int n) {
    for (int i = 0; i < n - 1; i++) {
        bool swapped = false;
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                int temp = arr[j];
                arr[j] = arr[j + 1];
                arr[j + 1] = temp;
                swapped = true;
            }
        }
        if (!swapped) break; // Early termination if no swaps occurred
    }
}

Explanation

The optimized Bubble Sort algorithm improves the basic version by checking if any swaps were made during a pass. If no swaps were made, the array is already sorted, and the algorithm can terminate early. This reduces unnecessary passes, improving efficiency for nearly sorted arrays. The time complexity in the best case (when the array is already sorted) is O(n).

Dry Run

Let's take an example: arr = {12, 11, 22, 25, 34, 64, 90} (almost sorted).

  • Initial Array: {12, 11, 22, 25, 34, 64, 90}
  • First Pass:
    • Compare 12 and 11 -> swap -> {11, 12, 22, 25, 34, 64, 90}
    • Compare 12 and 22 -> no swap -> {11, 12, 22, 25, 34, 64, 90}
    • Continue comparing remaining pairs -> no swaps -> {11, 12, 22, 25, 34, 64, 90}
  • Second Pass:
    • No swaps are made, so the algorithm terminates early.

Pros and Cons

Pros:

  • More efficient for nearly sorted arrays; early termination reduces unnecessary passes.

Cons:

  • Still O(n2) in the worst case, but offers better performance in practical scenarios.

Bidirectional Bubble Sort (Cocktail Shaker Sort)

void cocktailShakerSort(int arr[], int n) {
    bool swapped = true;
    int start = 0;
    int end = n - 1;

    while (swapped) {
        swapped = false;

        // Forward pass
        for (int i = start; i < end; i++) {
            if (arr[i] > arr[i + 1]) {
                int temp = arr[i];
                arr[i] = arr[i + 1];
                arr[i + 1] = temp;
                swapped = true;
            }
        }

        if (!swapped) break;

        swapped = false;
        end--;

        // Backward pass
        for (int i = end - 1; i >= start; i--) {
            if (arr[i] > arr[i + 1]) {
                int temp = arr[i];
                arr[i] = arr[i + 1];
                arr[i + 1] = temp;
                swapped = true;
            }
        }

        start++;
    }
}

Explanation

Bidirectional Bubble Sort, also known as Cocktail Shaker Sort, improves upon the basic Bubble Sort by sorting the array in both directions in each pass. This helps to quickly move large or small elements to their correct positions, reducing the total number of passes needed.

Dry Run

Let's take an example: arr = {64, 34, 25, 12, 22, 11, 90}.

  • Initial Array: {64, 34, 25, 12, 22, 11, 90}
  • First Forward Pass:
    • Similar to the first pass in basic Bubble Sort -> {34, 25, 12, 22, 11, 64, 90}
  • First Backward Pass:
    • Compare 64 and 11 -> swap -> {34, 25, 12, 22, 11, 64, 90}
    • Continue comparing remaining pairs -> no further swaps.
  • Subsequent Passes:
    • Forward and backward passes continue until the array is sorted.

Pros and Cons

Pros:

  • More balanced sorting, especially effective for nearly sorted arrays or those with large elements at the ends.

Cons:

  • Slightly more complex to implement; still O(n2) in the worst case.

Conclusion

Bubble Sort is an excellent starting point for understanding sorting algorithms, but it is not suitable for large datasets due to its quadratic time complexity. The optimized versions, such as early termination and bidirectional sorting, offer better performance in specific scenarios. However, for large datasets, more advanced algorithms like Quick Sort or Merge Sort are recommended.

By understanding and implementing these optimizations, you can choose the most efficient approach depending on the characteristics of your dataset, making your programs more efficient and responsive.
