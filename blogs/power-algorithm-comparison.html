Power Algorithm Comparison

Code

double power(double base, int exponent) {
        double result = 1.0;
        for (int i = 0; i < exponent; i++) {
            result *= base;
        }
        return result;
    }

Explanation

The basic power algorithm computes the power of a base raised to an exponent by multiplying the base exponent times. The time complexity of this algorithm is O(n), where n is the exponent.

Dry Run

Let's take an example: base = 2, exponent = 5.

  • Initialize result = 1.0.
  • Loop from i = 0 to i = 4 (5 iterations):
    • Iteration 1: result = 1.0 * 2 = 2.0
    • Iteration 2: result = 2.0 * 2 = 4.0
    • Iteration 3: result = 4.0 * 2 = 8.0
    • Iteration 4: result = 8.0 * 2 = 16.0
    • Iteration 5: result = 16.0 * 2 = 32.0

The final result is 32.0.

Pros

  • Simple to implement and understand.

Cons

  • Inefficient for large exponents due to linear time complexity.

Optimized Power Algorithm (Exponentiation by Squaring)

Code

double power(double base, int exponent) {
        if (exponent == 0) {
            return 1.0;
        }
        double half = power(base, exponent / 2);
        if (exponent % 2 == 0) {
            return half * half;
        } else {
            return half * half * base;
        }
    }

Explanation

The optimized power algorithm, also known as Exponentiation by Squaring, reduces the time complexity to O(log n). It divides the problem into smaller subproblems and uses the property:

  • If the exponent is even: an = (an/2)2
  • If the exponent is odd: an = a × (a(n−1)/2)2

Dry Run

Let's take an example: base = 2, exponent = 5.

  • Call power(2, 5):
    • Call power(2, 2):
      • Call power(2, 1):
        • Call power(2, 0): return 1.0
      • Return 1.0 * 1.0 * 2 = 2.0
    • Return 2.0 * 2.0 = 4.0
  • Return 4.0 * 4.0 * 2 = 32.0

The final result is 32.0.

Pros

  • Much more efficient for large exponents due to logarithmic time complexity.

Cons

  • Slightly more complex to implement.
Conclusion: The optimized power algorithm is significantly faster for large exponents and should be preferred over the basic power algorithm in such cases.
