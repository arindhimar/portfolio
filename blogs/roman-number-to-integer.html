Understanding Roman Numerals to Integer Conversion

Approach 1: Brute Force

int romanToInt(string s) {
        int result = 0;
        for (int i = 0; i < s.length(); i++) {
            if (s[i] == 'I') result += 1;
            else if (s[i] == 'V') result += 5;
            else if (s[i] == 'X') result += 10;
            else if (s[i] == 'L') result += 50;
            else if (s[i] == 'C') result += 100;
            else if (s[i] == 'D') result += 500;
            else if (s[i] == 'M') result += 1000;
        }
        return result;
    }

Explanation

The brute force approach involves iterating over the entire string and checking each character to see if it matches a Roman numeral. If it does, add the corresponding integer value to the result. This approach has a time complexity of O(n), where n is the length of the input string.

Dry Run

Index Character Result Explanation
0 M 1000 M = 1000, result += 1000
1 C 1100 C = 100, result += 100
2 M 2100 M = 1000, result += 1000
3 X 2110 X = 10, result += 10
4 C 2210 C = 100, result += 100

Pros and Cons

Pros:

  • Easy to implement and understand.

Cons:

  • Inefficient and slow for large input strings.

Approach 2: Hash Table

int romanToInt(string s) {
        unordered_map romanMap = {{'I', 1}, {'V', 5}, {'X', 10}, {'L', 50}, {'C', 100}, {'D', 500}, {'M', 1000}};
        int result = 0;
        for (int i = 0; i < s.length(); i++) {
            result += romanMap[s[i]];
        }
        return result;
    }

Explanation

The hash table approach involves creating a mapping of Roman numerals to their corresponding integer values and then iterating over the input string to look up the values in the hash table. This approach has a time complexity of O(n), where n is the length of the input string.

Dry Run

3 X 2110 Lookup X in map: 10, result += 10
4 C 2210 Lookup C in map: 100, result += 100
5 V 2215 Lookup V in map: 5, result += 5

Pros and Cons

Pros:

  • Faster than the brute force approach and easy to implement.

Cons:

  • Still has a time complexity of O(n) and may not be efficient for very large input strings.

Approach 3: Two-Pointer Technique

int romanToInt(string s) {
        unordered_map romanMap = {{'I', 1}, {'V', 5}, {'X', 10}, {'L', 50}, {'C', 100}, {'D', 500}, {'M', 1000}};
        int result = 0;
        for (int i = 0; i < s.length() - 1; i++) {
            if (romanMap[s[i]] < romanMap[s[i + 1]]) {
                result -= romanMap[s[i]];
            } else {
                result += romanMap[s[i]];
            }
        }
        result += romanMap[s.back()];
        return result;
    }

Explanation

The two-pointer technique involves iterating over the input string and comparing each character with the next one. If the current character is smaller than the next one, subtract its value from the result. Otherwise, add its value to the result. Finally, add the value of the last character to the result.

Dry Run

Index Character Next Character Result Explanation
0 M C 1000 M >= C, result += 1000
1 C M 900 C < M, result -= 100
2 M X 1900 M >= X, result += 1000
3 X C 1890 X < C, result -= 10
4 C V 1990 C >= V, result += 100
5 V - 1995 V is last, result += 5

Pros and Cons

Pros:

  • Efficient with a time complexity of O(n).
  • Reduces unnecessary additions and subtractions.

Cons:

  • More complex to understand compared to the other approaches.
